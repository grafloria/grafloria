Dashboard grid interaction — what I will build

GridStack's interaction model (verified against its v13 source), mapped mechanic-by-mechanic onto what Grafloria has today (each claim probed live on the running page), with the build plan on top. Section 1 is a working prototype of the exact model — the same integer math that will move into the engine. Everything is additive: a new layout type + a new kit module; nothing existing changes behaviour.

1Feel it first — the model, live (this exact math ports into the engine)

drag a tile anywhere · resize from the corner · double-click = pin/lock · Esc cancels a drag
What to notice while dragging: (a) the grey dashed placeholder always shows the exact drop result; (b) neighbours slide out of the way while you hover — your first comment — and they only budge when you cover >50% of them (the anti-jitter gate); (c) resizing wider pushes the next tile right-then-down and re-packs — your second comment; (d) the held tile follows the cursor 1:1 (transition-exempt) while everyone else glides at .3s; the drop itself is instant, into the placeholder; (e) a pinned tile is never pushed — others flow around it; (f) with float off, gravity packs tiles upward — no holes; toggle float to keep gaps.

3The build — a pack engine in the library, not demo JS

A · GridPackEngine — pure integer math (engine)

  • DOM-free, deterministic, cloneable (what-if a move before committing — GridStack's own trick, and a perfect fit for our TDD/mutation discipline).
  • Operations: moveCheck (with the >50% directional coverage gate), push (down, recursive, full-row sweep), swap (same-size neighbours), pack (gravity | float-with-memory), growRows.
  • Locked tiles: never pushed, jumped over — wired to our existing authoritative lock.
  • Cell state lives in the existing GridItemConfig — no new persistence, save/load already round-trips it.
  • Registered as a NEW group layout type 'pack' beside flexbox/grid/columns — zero edits to the existing layout paths or their specs.
new: libs/engine/src/layout/grid-pack/… + table-driven specs (every rule mutation-proven)

B · Drag & resize session (element kit)

  • bindDashboardGestures(api, group): pointer-capture loop over the HTML hosts (pointer events — the binder eats compat mouse events; learned that the hard way).
  • Drag: held tile = transition-exempt ghost tracking the cursor in world coords (our camera math replaces GridStack's position:fixed hack); a placeholder tile enters the engine in its stead; every cell-crossing runs moveCheck → neighbours reflow live with the glide we shipped today.
  • Resize: corner handle → fluid pixel preview; each cell-step change runs the same pipeline → live push right/down; min/max spans clamped continuously.
  • Escape restores the pre-drag snapshot (engine keeps it, like GridStack's saveInitial).
  • Gesture end = ONE BatchCommand (SetGridItem/Move/Resize for every displaced tile) — single-step undo per gesture, which GridStack itself cannot do.
new: libs/element/src/lib/dashboard-kit/… (same shape as diagram-kit: spec + binder + typed handles)

C · Demo becomes the showcase

  • dashboard-builder.html switches its views to 'pack' + the kit binder; the interim release-settle module is retired (it was scaffolding for exactly this).
  • Palette upgraded to drag-in (clone follows cursor, placeholder appears on board entry — first placement skips the 50% gate, per GridStack) — click-to-add stays.
  • Trash-corner removal like web2 (drag a tile out → dim → release deletes, undoable).

D · Responsive columns (phase 2)

  • columnWidth-driven column count from container width (the responsive_none demo's model), with GridStack's per-column-count layout cache so narrow→wide round-trips losslessly and saving on mobile saves the desktop layout.
  • RTL rides the same engine later — the cell model is direction-agnostic; only the px mapping mirrors.

4Nothing breaks — the guarantees

  • New layout type, not a rewrite: 'pack' is registered beside flexbox/grid/columns. The existing paths, their integration specs (asserted pixel positions) and all 102 other demos are untouched.
  • Same storage: cell state = the existing GridItemConfig; serialize/round-trip already proven. No migration.
  • Same undo contract: every gesture commits through the CommandManager — stronger than GridStack, kept.
  • Today's fixes carry forward: chromeless containers, connectable:false, the glide rule (the pack loop drives MORE reflows through the SAME animation), board growth, pin.
  • Gates before merge: engine specs mutation-proven per rule; new e2e teeth — placeholder-truthfulness (drop lands exactly where the placeholder stood), live-push-during-hover, resize-pushes-neighbour, Esc-restore; full gallery + interaction + goldens green.
