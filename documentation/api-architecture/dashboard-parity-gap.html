Plan vs delivered — dashboard parity, measured

Capability-by-capability comparison of the plan prototype against the delivered demos/dashboard/dashboard-builder.html. Every row was PROBED with a real mouse on both pages (not asserted from reading code); the six user-review rounds are folded in. Verdicts: SAME behaves identically · FIXED Rn diverged, fixed in review round n · DELIBERATE intentionally different (product reasons, listed openly).

5How this stays true

The battery is the gate: node demos/e2e/dashboard-scenarios-run.mjs — 24 scenarios, real mouse, placeholder-truth at every drop, a screenshot at every stage. Every behaviour in §1 has a scenario; a regression turns a row red with its geometry evidence printed. The five review rounds each added the scenario that would have caught them (s10 overshoot-survival, s13 grow-camera, s21 strip growth, s23 float, s24 alignment).
